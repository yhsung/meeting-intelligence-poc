# Meeting Intelligence POC — 提案文件

> 將過去任意幾次會議轉化為可對話的 AI 知識庫，讓任何人都能隨時「進入」這些會議，獲取精華洞見。

## 背景與目標

- 週一早上 GM Demo
- 展現 BU 團隊生產力提升，以及 agentic coding 快速將 idea 變現的能力
- 使用現成 YouTube 影片作為資料來源，無特定技術棧限制
- 適用於各種會議類型

---

## 問題陳述

| 痛點 | 現況 | 代價 |
|------|------|------|
| 資訊孤島 | 沒參加的人只能看靜態 meeting minutes | 關鍵 context 流失，需要人工補充說明 |
| 時間成本 | 要吸收內容只能重看完整錄影 | 一小時會議 = 一小時補課 |
| 知識沉澱 | 決策理由、討論過程不在 minutes 裡 | 幾週後連當事人都忘了「為什麼這樣決定」 |

---

## 解決方案

### 核心流程

```
YouTube 會議錄影
      │
      ▼
[自動轉錄 + 摘要]  ──→  結構化知識片段
      │
      ▼
用戶選定「哪幾次會議」
      │
      ▼
組建 AI 知識庫 (RAG)
      │
      ▼
任何人皆可對話提問
```

### 用戶體驗

1. **選擇會議** — 貼入 YouTube 會議連結，或從列表中勾選過去幾場
2. **建立知識庫** — AI 自動轉錄、切片、建立索引（< 2 分鐘）
3. **開始對話** — 輸入問題，如「這個 Q2 roadmap 當時為什麼選方案 B 而不是 A？」
4. **溯源回答** — AI 回應並標記來源（哪場會議、幾分鐘處）

---

## 技術架構

| 元件 | 工具 | 理由 |
|------|------|------|
| 影片轉文字 | YouTube Transcript API / Whisper | 有字幕直接取；無字幕 Whisper 轉 |
| 向量搜尋 | ChromaDB / FAISS (本地) | 零成本，快速部署 |
| AI 對話 | Claude API (claude-sonnet-4-6) | 長 context，理解力強 |
| 前端 | Streamlit / Gradio | 1 天內做出可 Demo 的 UI |
| 會議選擇器 | Checkbox UI | 模擬多會議知識庫組合 |

---

## Demo 腳本（週一 GM 版本）

**準備材料：** 3 支 YouTube 錄影（模擬公司會議）

**Demo 流程（10 分鐘）：**

1. 展示「會議資料庫」介面 — 列出 5 場過去會議，勾選 3 場
2. 點擊「建立知識庫」，等待處理完成
3. 現場提問示範：
   - _「這三次討論中，大家對 X 方向最大的疑慮是什麼？」_
   - _「有沒有提到 timeline 或 deadline？」_
   - _「如果我是新人，我需要先了解哪些背景？」_
4. 展示來源溯源：AI 回答後顯示「來自第 2 場會議 14:32」

## 價值主張

### 對個人
- 補課時間從 **1 小時 → 5 分鐘**
- 不再需要騷擾同事「你幫我 recap 一下」

### 對團隊
- 決策理由有跡可查，不再憑記憶爭論
- 跨時區/跨部門成員零障礙同步

### 對公司
- 知識資產化：每場會議都是可查詢的 institutional knowledge
- 展示 **agentic coding 落地速度**：從 idea 到 GM demo，一個週末

---

## 商業延伸潛力

- 與現有內部系統整合（Teams / Zoom / Confluence）
- 跨會議主題聚合（「過去 6 個月所有討論 roadmap 的會議」）
- 自動生成 action item 追蹤
- 存取控制（哪些人可查哪些會議）

## 這個提案本身就是最好的示範
